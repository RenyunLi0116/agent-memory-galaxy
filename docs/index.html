From scattered traces to one galaxy.

Plain files and static HTML — here’s how one day of agent work becomes one graph.

STEP 01 · IN EVERY PROJECT

Agents write memory as they work

Each project keeps an agent_memory.md: what changed, why, and which files. Agents read it when a session starts and append after meaningful changes.

STEP 02 · ON EVERY MACHINE

Each machine contributes a fragment

contribute.sh scans reviewed notes and safe session metadata — never raw conversations — into fragments/<machine>.json.

STEP 03 · ON THE AGGREGATOR

One aggregator merges the graph

Shared files, datasets, models, and servers connect projects across machines. A gold reference skeleton emerges between project hubs.

STEP 04 · LIVE, EVERY FEW MINUTES

Working agents light up red

Agents that are mid-task show a red pulse plus a one-line status of what they’re working on — so you and every other agent see the overlap before it happens.

STEP 05 · ONLY IF YOU PUBLISH

Encryption before anything leaves

Plaintext graphs stay local or in a private hub. A public Pages deploy ships the viewer shell plus ciphertext only, unlocked client-side.

Install with two Claude Code commands.

The repo ships a Claude Code plugin. The skill sets up a private hub, pulls in each machine, merges their memory into one graph, and runs a privacy review before anything is published.

Public framework. Private memory.

Working solo, your agent_memory.md and fragments never leave your machine — nothing phones home. GitHub Pages carries only the open framework and this demo.

On a team, distilled fragments — safe session metadata, never raw conversations — sync as plaintext inside your private GitHub repo, readable only by the collaborators you add. The only thing that ever leaves that repo is a public Pages deploy, and that ships AES-256-GCM ciphertext, unlocked in the browser.
