Three ways the day falls apart.

Run agents on more than one machine and you already know all three.

PAIN 01

Nobody remembers who did what

Each agent leaves its trail in a different scrollback on a different machine. By the next morning, "why did this change?" has no answer.

PAIN 02

Duplicate work, hours apart

Two agents chase the same bug on two machines. Neither knows a fix already exists, so the same afternoon gets spent twice.

PAIN 03

No collaboration network

Agents cannot see each other, so there is no handoff: configs get overwritten, runs get orphaned, and context dies with the session.

From scattered traces to one galaxy.

Plain files and static HTML - no daemon, no database, no build step. This is the path a day of agent work takes.

STEP 01 - IN EVERY PROJECT

Agents write memory as they work

Each project keeps an agent_memory.md: what changed, why, and which files. Agents read it when a session starts and append after meaningful changes.

STEP 02 - ON EVERY MACHINE

Each machine contributes a fragment

contribute.sh scans reviewed notes and safe session metadata - never raw conversations - into fragments/<machine>.json.

STEP 03 - ON THE AGGREGATOR

One aggregator merges the graph

Shared files, datasets, models, and servers connect projects across machines. A gold reference skeleton emerges between project hubs.

LIVE - EVERY FEW MINUTES

Working agents light up red

Agents currently on the clock carry a red pulse and a doing line that says what they are on - visible to you and to every other agent before work overlaps.

STEP 05 - ONLY IF YOU PUBLISH

Encryption before anything leaves

Plaintext graphs stay local or in a private hub. A public Pages deploy ships the viewer shell plus ciphertext only, unlocked client-side.

Public framework. Private memory.

GitHub Pages is not private access control. Public pages carry the framework and fake demo data. Real fragments and plaintext graphs belong in a private hub or local machine.

Contributor

Writes `fragments/<machine>.json` in a private hub. Does not need encryption passwords and should not touch `docs/galaxy/`.

Aggregator

Merges all fragments, builds local `standalone.html`, and optionally creates `docs/galaxy/graph.enc.json` with strong passwords.
