# Portability

Agent Memory Galaxy is intentionally simple: Python scripts produce a JSON graph, and static HTML renders it. The system should survive new machines, new agents, and new hosting surfaces.

## Moving the Aggregator

On the new aggregator:

```bash
git clone git@github.com:your-org/<private-hub>.git
cd <private-hub>
./update.sh
```

If you publish an encrypted viewer, recreate `.amg_password` and `.amg_password2` locally. Password files are not committed.

## Moving Contributors

Clone the private hub and run:

```bash
./contribute.sh <new-machine-name> <claude|codex|cursor|human> [scan-root]
```

Use a new unique machine name unless you intentionally want to replace an existing fragment.

## Publishing Options

| Output | Path | Policy |
|---|---|---|
| Local graph | `graph.json` | Plaintext, gitignored |
| Local viewer | `standalone.html` | Plaintext, gitignored |
| Public landing page | `docs/index.html` | Safe, no real data |
| Synthetic demo | `docs/demo/` | Safe if generated by `scripts/build-public-demo.py` |
| Encrypted viewer | `docs/galaxy/index.html` + `docs/galaxy/graph.enc.json` | Publishable with strong passwords |

The public template ignores private artifacts by default. In a private hub, use `AMG_TRACK_PRIVATE=1 ./update.sh` when you intentionally want Git to track real fragments, presence, or encrypted viewer data.

## Password Loss

If encrypted viewer passwords are lost, regenerate new password files and rerun `encrypt.py` or `update.sh`. Old ciphertext cannot be decrypted without the old passwords.

## Agent Independence

The durable interface is `agent_memory.md`, not a specific AI product. Claude, Codex, Cursor, and humans can all write the same markdown format and contribute to the same graph.
